<task type="auto">
  <name>Task 1: Build hierarchical tool/package payload for CLI list</name>
  <files>src/bioimage_mcp/bootstrap/list.py</files>
  <action>
- Keep MCP API behavior unchanged. This work is CLI-only.
- In `list_tools`, transform loaded manifests into a hierarchical payload with this exact JSON shape:
  - Top-level output: `{"tools": [ToolInfo, ...]}`
  - ToolInfo keys (all required):
    - `id` (short tool id; drop `tools.` prefix if present)
    - `tool_version` (existing manifest tool_version)
    - `library_version` (set to null in this plan; real resolution comes in 17-02)
    - `status` ("installed" if env exists, else "partial")
    - `function_count` (int)
    - `packages` (list of PackageInfo)
  - PackageInfo keys (all required):
    - `id` (package id under tool)
    - `library_version` (set to null in this plan)
    - `function_count` (int)
- Package grouping rules (deterministic):
  - Let `tool_short_id` be tool `id` with `tools.` stripped.
  - For each function id string `fn_id`:
    - If `fn_id` starts with `{tool_short_id}.`, strip that prefix and the following dot.
    - Else keep `fn_id` as-is.
    - `package_id` is the first segment before the next dot.
  - Group function counts by `package_id` and sort packages lexicographically by `id`.
- Keep `--tool` filter behavior, but update it to match BOTH:
  - `--tool base` -> matches tool id `base`
  - `--tool tools.base` -> matches tool id `base`
  </action>
  <verify>python -m bioimage_mcp.cli list --json >/dev/null</verify>
  <done>
`bioimage-mcp list --json` returns the new hierarchical schema and includes `packages` per tool.
  </done>
</task>
