<task type="auto">
  <name>Task 1: Implement lockfile-first library version resolution</name>
  <files>src/bioimage_mcp/bootstrap/list.py</files>
  <action>
- Implement a small helper (in this module unless a new helper file is clearly cleaner) that resolves versions in this order:
  1) Parse `envs/<env_id>.lock.yml` (conda-lock YAML) and find the matching conda package name.
     - Use `yaml.safe_load`.
     - Prefer entries whose `platform` matches the current platform (linux-64/osx-arm64/win-64). If platform detection is non-trivial, use a conservative mapping based on `sys.platform` + `platform.machine()`.
  2) If no lockfile match and an env manager executable exists, run: `<manager_exe> list -n <env_id> --json` and parse the JSON list to find the package version.
  3) If still unresolved, return `None` (rendering will fall back to `tool_version`).

- Hardcode the mapping needed by Phase 17 CONTEXT (package IDs -> conda package names):
  - base: scipy->scipy, phasorpy->phasorpy, skimage->scikit-image, pandas->pandas, xarray->xarray
  - cellpose tool primary library: cellpose->cellpose (apply to all its packages)
  - trackpy tool primary library: trackpy->trackpy (apply to all its packages)
  - stardist tool primary library: stardist->stardist (apply to all its packages)
  - tttrlib tool primary library: tttrlib->tttrlib

- Wire resolved versions into the hierarchical payload:
  - ToolInfo.library_version: set to resolved primary library version when applicable (cellpose/trackpy/stardist/tttrlib), else null.
  - PackageInfo.library_version:
    - For tools with a primary library version, set packages to the same version.
    - For base subpackages, resolve per-package when mapping exists; otherwise null.
- Update the table rendering so the Version column prefers `library_version` when present, otherwise falls back to `tool_version`.
  </action>
  <verify>python -m bioimage_mcp.cli list --json >/dev/null</verify>
  <done>
At least one real library version (from a lockfile) appears in CLI list JSON/table output when the corresponding `envs/<env_id>.lock.yml` exists.
  </done>
