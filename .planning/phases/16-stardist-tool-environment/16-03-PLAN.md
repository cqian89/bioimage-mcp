  <name>Task 1: Add StarDist ops helpers (I/O + normalization + writers)</name>
  <files>

  <action>
- Create an ops module mirroring the Cellpose pattern:
  - `ops/utils.py`: helpers to convert artifact `uri` to `Path`, load BioImageRef via `bioio.BioImage`, and write OME-Zarr labels via `bioio_ome_zarr.writers.OMEZarrWriter`.
  - `ops/predict.py`: implement a function that:
    - Accepts inputs dict containing at least `image` (BioImageRef) and a `model` ObjectRef URI OR parameters to construct a model.
    - Loads image data using bioio, preserves native axes (no TCZYX padding).
    - Normalizes input using `csbdeep.utils.normalize` (no hand-rolled normalization).
    - Calls `StarDist2D/StarDist3D.predict_instances`.
    - Writes `labels` as OME-Zarr directory under work_dir and returns an artifact dict shaped like Cellpose outputs:
      - `type: LabelImageRef`, `format: OME-Zarr`, `storage_type: zarr-temp`, include `metadata.shape/dtype/dims`.
    - Serializes `details` to JSON on disk and returns `NativeOutputRef` with `format: stardist-details-json` (or similar stable format string).
- Keep TensorFlow noise down by setting `TF_CPP_MIN_LOG_LEVEL=2` in the process (entrypoint-level is fine).
  </action>
  <verify>python -m compileall tools/stardist/bioimage_mcp_stardist/ops</verify>
  <done>
StarDist ops exist and compile; predict can write OME-Zarr labels and a JSON details artifact.
  </done>
