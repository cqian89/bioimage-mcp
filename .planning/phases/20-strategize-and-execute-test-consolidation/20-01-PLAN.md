  <name>Task 1: Define smoke tier markers and move requires_env skipping to a global fixture</name>
  <files>
pytest.ini
tests/conftest.py
tests/smoke/conftest.py
  </files>
  <action>
Update marker definitions and env-gated skip behavior to match Phase 20 decisions.

1) Update `pytest.ini` markers:
- Add `smoke_pr`: PR-gating smoke tier.
- Add `smoke_extended`: extended/nightly smoke tier.
- Keep `smoke_minimal`.
- Keep `smoke_full` defined only as a deprecated marker (do not rely on it for selection after Phase 20).

2) Implement global env gating in `tests/conftest.py` (applies to ALL test directories, not just `tests/smoke/`):
- Add an autouse fixture `check_required_env`.
- Behavior:
  - If a test has `@pytest.mark.requires_env("bioimage-mcp-...")` and the env is missing/unavailable, then:
    - `warnings.warn(...)` with an actionable message including the install command form:
      - Prefer `bioimage-mcp install <tool_name>` where `<tool_name>` comes from env name `bioimage-mcp-<tool_name>`.
    - Then `pytest.skip(...)` with a short reason.
  - Continue supporting legacy convenience markers (`requires_base`, `requires_cellpose`, `requires_stardist`) as aliases (no new tests should be written using them).

Implementation constraints:
- Cache environment availability checks for the duration of the test session to avoid repeated `conda run ...` calls.
- Do not hard-fail if conda/mamba is not installed; treat env as unavailable and skip.

3) Update `tests/smoke/conftest.py` smoke suite selection:
- Replace the single `minimal_mode` boolean with a simple mode enum:
  - `minimal` (default): run only `smoke_minimal` items when invoking `pytest tests/smoke` without marker filtering.
  - `pr`: run `smoke_minimal` + `smoke_pr`.
  - `extended`: run `smoke_minimal` + `smoke_pr` + `smoke_extended`.
- CLI options:
  - Add `--smoke-pr` to set mode `pr`.
  - Add `--smoke-extended` to set mode `extended`.
  - Keep `--smoke-full` as an alias for `--smoke-extended` (backward compatibility).
- Update the skip fixture so:
  - mode=minimal skips any item marked `smoke_pr` or `smoke_extended`.
  - mode=pr skips any item marked `smoke_extended`.
  - mode=extended does not skip based on tier markers.

Remove the existing env gating fixture from `tests/smoke/conftest.py` (it will be handled globally).
  </action>
  <verify>pytest tests/smoke/test_smoke_structure.py -m smoke_minimal -q</verify>
  <done>
- `pytest --markers` shows smoke_minimal/smoke_pr/smoke_extended.
- Any `requires_env(...)` test skips (with a warning) when env is missing, regardless of test directory.
- `pytest tests/smoke` defaults to minimal behavior; `pytest tests/smoke --smoke-pr` expands to include smoke_pr.
  </done>
</task>
