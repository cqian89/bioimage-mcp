Update smoke test tier markers and dataset markers to match the new policy.

Tiering rules:
- `smoke_minimal`: keep existing minimal tests as-is.
- `smoke_pr`: PR-gating smoke tier MUST include:
  - ONE representative base image-filter equivalence theme (choose: `tests/smoke/test_equivalence_skimage.py::test_skimage_gaussian_equivalence`).
  - Representative optional tool env equivalence: Cellpose + Trackpy.
- `smoke_extended`: everything else that used to be `smoke_full`.

Concrete changes:
1) For ALL tests currently marked `@pytest.mark.smoke_full`, replace with either `smoke_pr` or `smoke_extended`:
- Mark as `smoke_pr`:
  - `tests/smoke/test_equivalence_skimage.py::test_skimage_gaussian_equivalence`
  - `tests/smoke/test_equivalence_cellpose.py::test_cellpose_equivalence`
  - `tests/smoke/test_equivalence_trackpy.py::test_trackpy_locate_equivalence`
- Mark ALL other former `smoke_full` tests as `smoke_extended`.

2) Ensure every `test_equivalence_*.py` test function has exactly one dataset marker:
- Base/skimage/scipy/matplotlib/pandas/xarray/phasorpy: `@pytest.mark.uses_minimal_data`
- Trackpy equivalence: `@pytest.mark.requires_lfs_dataset` (uses vendored dataset under `datasets/trackpy-examples/...`).
- Stardist equivalence: choose dataset marker based on its input path; default to `requires_lfs_dataset` if it uses non-synthetic data.

3) Remove deprecated convenience env markers usage inside smoke equivalence tests:
- In `tests/smoke/test_equivalence_stardist.py`, remove `@pytest.mark.requires_stardist` and keep `@pytest.mark.requires_env("bioimage-mcp-stardist")`.

4) Keep directory/marker intent aligned:
- Tests under `tests/smoke/` should have one of the smoke tier markers (smoke_minimal, smoke_pr, smoke_extended).
  - If a file contains a mix of minimal and extended tests (e.g. `test_smoke_scipy_submodules.py`), keep per-test markers.
