  <name>Task 1: Compute and enforce env_lock_hash + source_hash in describe_function caching, and sync functions table metadata</name>
  <files>src/bioimage_mcp/api/discovery.py</files>
  <action>
Fix `DiscoveryService.describe_function()` so cache invalidation keys are used and list/describe stay consistent.

Implementation requirements:

1) Add small, test-friendly helpers inside `src/bioimage_mcp/api/discovery.py` (module-level functions) so tests can monkeypatch them:
- `_find_project_root(start: Path) -> Path | None`: copy the heuristic from `src/bioimage_mcp/registry/loader.py` (walk up a few parents looking for `envs/` or `pyproject.toml`).
- `_compute_env_lock_hash(project_root: Path | None, env_id: str) -> str | None`:
  - If `project_root` is None or `envs/{env_id}.lock.yml` missing, return None.
  - Else return `sha256(file_text).hexdigest()[:16]`.
- `_compute_source_hash(manifest: ToolManifest, *, module: str | None, callable_name: str | None, project_root: Path | None) -> str | None`:
  - If module/callable_name missing, return None.
  - Use `inspect_module(module, search_paths)` with search_paths:
    - Always include `manifest.manifest_path.parent`.
    - If `project_root` exists and has `tools/`, include `project_root / "tools"`.
  - Find the matching `StaticCallable` by `name == callable_name`.
  - If found and `source` present, return `callable_fingerprint(source)`.
  - Otherwise return None.

2) In the meta.describe enrichment block (where `get_cached_schema()`/`upsert_schema_cache()` are used):
- Compute `project_root = _find_project_root(manifest.manifest_path.parent)`.
- Compute `env_lock_hash = _compute_env_lock_hash(project_root, manifest.env_id)`.
- Compute `source_hash = _compute_source_hash(...)` using:
  - module: prefer `node.module` (already in DB/list), else `function_def.module`.
  - callable_name: prefer `payload.get("name")`, else last segment of `fn_id`.

- Call `self._index.get_cached_schema(..., env_lock_hash=env_lock_hash, source_hash=source_hash)`.
- On cache hit: use cached params_schema + introspection_source.
- On cache miss: call `execute_tool(meta.describe)` and parse via `parse_meta_describe_result()`; then call:
  `self._index.upsert_schema_cache(..., env_lock_hash=env_lock_hash, source_hash=source_hash, callable_fingerprint=result.get("callable_fingerprint") or source_hash)`.

3) Ensure list/describe consistency:
- After determining the final `params_schema` and `introspection_source`, update the `functions` table row via `self._index.upsert_function(...)` using the existing payload values (tool_id/name/description/tags/inputs/outputs/module/io_pattern) but with:
  - `params_schema` set to the final schema returned by describe
  - `introspection_source` set to the final introspection_source

This makes `list_tools()` (which reads the functions table) reflect the same `introspection_source` as describe.

4) Remove any remaining dependency on schema_cache.json within describe_function:
- Do not read/write `config.schema_cache_path` when `config.schema_cache_use_db` is true (default).
  </action>
