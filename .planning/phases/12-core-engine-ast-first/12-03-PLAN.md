  <files>src/bioimage_mcp/registry/engine.py</files>
  <action>
Create `DiscoveryEngine` responsible for producing `ToolManifest.functions` entries from:
- Static manifest-defined functions (pass-through, but normalize fn_id/module to full module paths).
- `dynamic_sources` (AST-first using the static inspector from Plan 12-01).

Engine behavior (from Phase 12 decisions):
- Runtime overrides AST on conflicts; full override allowed.
- Fallback triggers on any mismatch between AST-derived schema and runtime-derived schema.
- Record `introspection_source` per function (e.g., `ast`, `runtime:meta.describe`).
- If runtime fallback fails, skip the callable and record the error (log-only).
- Skipped callables do not appear in list/describe.

Implementation notes:
- Use `execute_tool` + `meta.describe` against the tool entrypoint for runtime fallback.
- Use `parse_meta_describe_result()` for normalized output.
- Ensure `params_schema` emitted by the engine is deterministic (use the normalizer from Plan 12-01).
- Enforce artifact separation: omit artifact ports from `params_schema` even on fallback.

Avoid introducing new global caches in this plan; rely on existing per-request behavior (caching is handled later).
