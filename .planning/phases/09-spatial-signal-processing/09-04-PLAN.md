Implementation notes:
- Inherit from `ScipyNdimageAdapter` to reuse `_load_image` and `_save_image` helpers.
- Use `PandasAdapterForRegistry._save_table(...)` to persist spectral outputs as TableRef.
- For convolution/correlation, accept the kernel as the second artifact input (`input_1` per `IOPattern.BINARY`).
- Kernel shape handling: if `kernel.ndim < image.ndim`, left-pad kernel with leading singleton dims so that `kernel.ndim == image.ndim` (apply kernel over the trailing axes).

1) `scipy.signal.fftconvolve`:
- Call `scipy.signal.fftconvolve(image, kernel, mode=mode)`.
- Save output via `_save_image(...)` preserving the input axes/metadata (match ScipyNdimageAdapter behavior: infer axes from input metadata when available).

2) `scipy.signal.correlate`:
- Call `scipy.signal.correlate(image, kernel, mode=mode, method="fft")` (or use `fftconvolve(image, kernel[::-1], ...)` if needed).
- Save output via `_save_image(...)`.

3) `scipy.signal.periodogram` (IOPattern.ANY_TO_TABLE):
- Load a 1D numeric series from the single input artifact:
  - If `TableRef` or `ObjectRef` holding a DataFrame: use `PandasAdapterForRegistry()._load_table(...)`, then choose `params["column"]` if present else auto-select first numeric column.
  - If `BioImageRef`: use `_load_image(...)`, then `x = np.squeeze(arr)` and require `x.ndim == 1`.
- Call `scipy.signal.periodogram(x, fs=fs, scaling=scaling)`.
- Persist as `TableRef` via `PandasAdapterForRegistry()._save_table(df, name="periodogram", work_dir=work_dir)` where `df` has columns: `frequency`, `power`.

4) `scipy.signal.welch` (IOPattern.ANY_TO_TABLE):
- Load a 1D numeric series from the single input artifact using the same rules as `periodogram`.
- Call `scipy.signal.welch(x, fs=fs, nperseg=nperseg, noverlap=noverlap, scaling=scaling)`.
- Persist as `TableRef` via `PandasAdapterForRegistry()._save_table(df, name="welch", work_dir=work_dir)` where `df` has columns: `frequency`, `psd`.
