In `src/bioimage_mcp/registry/dynamic/adapters/scipy_signal.py`, implement `execute()` for the four curated wrapper fn_ids.

Implementation notes:
- Inherit from `ScipyNdimageAdapter` to reuse `_load_image`, `_save_image`, and `_save_json` helpers.
- For convolution/correlation, accept the kernel as the second artifact input (`input_1` per `IOPattern.BINARY`).
- Kernel shape handling: if `kernel.ndim < image.ndim`, left-pad kernel with leading singleton dims so that `kernel.ndim == image.ndim` (apply kernel over the trailing axes).

1) `scipy.signal.convolve_image`:
- Call `scipy.signal.fftconvolve(image, kernel, mode=mode)`.
- Save output via `_save_image(...)` preserving the input axes/metadata (match ScipyNdimageAdapter behavior: infer axes from input metadata when available).

2) `scipy.signal.correlate_image`:
- Call `scipy.signal.correlate(image, kernel, mode=mode, method="fft")` (or use `fftconvolve(image, kernel[::-1], ...)` if needed).
- Save output via `_save_image(...)`.

3) `scipy.signal.periodogram_table`:
- Load a 1D numeric series from the `table` input:
  - use `PandasAdapterForRegistry()._load_table(...)`
  - choose `params["column"]` if present, else auto-select first numeric column
- Call `scipy.signal.periodogram(x, fs=fs, scaling=scaling)`.
- Save a `TableRef` with columns: `frequency`, `power`.

4) `scipy.signal.welch_table`:
- Load series as above.
- Call `scipy.signal.welch(x, fs=fs, nperseg=nperseg, noverlap=noverlap, scaling=scaling)`.
- Save a `TableRef` with columns: `frequency`, `psd`.
