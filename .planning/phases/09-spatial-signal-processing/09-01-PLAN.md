  <files>src/bioimage_mcp/registry/dynamic/models.py, src/bioimage_mcp/registry/loader.py, tests/unit/registry/test_loader_io_patterns.py</files>
  <action>
Add new `IOPattern` members in `src/bioimage_mcp/registry/dynamic/models.py`:
- `TABLE_TO_OBJECT`: one table-like artifact in, ObjectRef out
- `OBJECT_AND_TABLE_TO_JSON`: ObjectRef + table-like artifact in, JSON out

Add a flexible 1D-signal input pattern for API-first scipy.signal spectral functions:
- `ANY_TO_TABLE`: one artifact in (`TableRef` / `BioImageRef` / `ObjectRef`), TableRef out

And add reusable file-output patterns for array-like outputs:
- `TABLE_TO_FILE`: one table-like artifact in, NativeOutputRef out
- `TABLE_PAIR_TO_FILE`: two table-like artifacts in, NativeOutputRef out

Then extend `src/bioimage_mcp/registry/loader.py:_map_io_pattern_to_ports()`:

- For `TABLE_TO_OBJECT`:
  - inputs: `table` (artifact_type: `["TableRef", "ObjectRef"]`)
  - outputs: `object` (artifact_type: `"ObjectRef"`)

- For `OBJECT_AND_TABLE_TO_JSON`:
  - inputs: `object` (artifact_type: `"ObjectRef"`) and `table` (artifact_type: `["TableRef", "ObjectRef"]`)
  - outputs: `output` (artifact_type: `"ScalarRef"`)

- For `TABLE_TO_FILE`:
  - inputs: `table` (artifact_type: `["TableRef", "ObjectRef"]`)
  - outputs: `output` (artifact_type: `"NativeOutputRef"`)

- For `TABLE_PAIR_TO_FILE`:
  - inputs: `table_a` and `table_b` (artifact_type: `["TableRef", "ObjectRef"]`)
  - outputs: `output` (artifact_type: `"NativeOutputRef"`)

- For `ANY_TO_TABLE`:
  - inputs: `input` (artifact_type: `["TableRef", "BioImageRef", "ObjectRef"]`)
  - outputs: `table` (artifact_type: `"TableRef"`)

Note: keep JSON outputs mapped as `ScalarRef` to preserve existing schema compatibility (the runtime returns JSON payloads using `NativeOutputRef` but the manifest schema uses `ScalarRef` for JSON-like outputs).

Update `tests/unit/registry/test_loader_io_patterns.py` by adding assertions for the new patterns.
