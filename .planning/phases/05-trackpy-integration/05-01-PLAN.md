Create the tool pack directory structure: `tools/trackpy/bioimage_mcp_trackpy/`

**manifest.yaml** - Follow `tools/base/manifest.yaml` pattern:
```yaml
manifest_version: "1.0"
tool_id: tools.trackpy
tool_version: "0.1.0"
name: Trackpy Particle Tracking Toolkit
description: Particle tracking and trajectory analysis using trackpy library.

env_id: bioimage-mcp-trackpy
entrypoint: bioimage_mcp_trackpy/entrypoint.py
python_version: "3.12"
platforms_supported:
  - linux-64
  - osx-arm64
  - win-64

# Dynamic sources will be added in Plan 02 when adapter is ready
dynamic_sources: []

# Explicit functions will be minimal - most come from dynamic discovery
functions: []
```

**__init__.py** - Simple package marker:
```python
"""Trackpy tool pack for bioimage-mcp."""
from __future__ import annotations

__version__ = "0.1.0"
```

**entrypoint.py** - NDJSON worker entrypoint:
Follow the pattern from `tools/base/bioimage_mcp_base/entrypoint.py` but simplified for trackpy.

Key elements:
1. Path setup (add src/ and tools/ to sys.path)
2. Constants: TOOL_VERSION, TOOL_ENV_NAME, DYNAMIC_FN_PREFIXES
3. FN_MAP (initially empty, will be populated by dynamic dispatch)
4. Memory artifact storage (global _MEMORY_ARTIFACTS dict)
5. Worker initialization (_initialize_worker function)
6. Main NDJSON loop that handles: execute, shutdown commands
7. Dynamic dispatch to TrackpyAdapter for trackpy.* functions

The entrypoint should:
- Send ready handshake on startup: `{"command": "ready", "version": "0.1.0"}`
- Route unknown fn_ids (trackpy.*) to dynamic_dispatch
- Handle graceful shutdown with memory cleanup

Import the dispatch_dynamic function from bioimage_mcp_base.dynamic_dispatch for now (adapter integration comes in Plan 02).
