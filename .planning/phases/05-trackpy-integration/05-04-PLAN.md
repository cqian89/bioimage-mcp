<task type="auto">
  <name>Enforce worker kill on read errors</name>
  <files>src/bioimage_mcp/runtimes/persistent.py</files>
  <action>
    Modify `WorkerProcess.execute` to ensure worker is killed if `read_response` fails.
    
    Current issue: If `readline` fails (e.g. decoding error) in the thread, `error` is captured, but the worker is NOT killed in the main thread before raising. This leaves the worker alive but the client state desynchronized (ordinal mismatch on next req).
    
    Fix:
    In the `if timeout_seconds is not None:` block, after `reader_thread.join(...)`:
    1. Check `if error:`
    2. If true:
       - Log the error.
       - Kill the worker (`self._process.kill()`, `self._process.wait()`).
       - Update state to `TERMINATED`.
       - Raise the error.
       
    This ensures that any communication breakdown results in a fresh worker for the next request.
  </action>
  <verify>
    Review `persistent.py` to ensure `if error:` block includes termination logic similar to the timeout block.
  </verify>
  <done>
    Worker is terminated on any read exception.
  </done>
</task>
