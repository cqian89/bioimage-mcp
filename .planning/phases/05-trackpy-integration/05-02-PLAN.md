**Key design decisions from CONTEXT.md:**
- Expose full trackpy API (exclude underscore-prefixed functions)
- Runtime introspection (dynamic discovery)
- Graceful degradation if introspection fails
- Function IDs follow trackpy API path (e.g., `trackpy.locate`, `trackpy.link`)
- Schemas permissive when docs ambiguous
- Include default values from Python signatures
- Attach docstring-derived descriptions to schema fields
