<task type="auto">
  <name>Scipy adapter execute: named inputs, dtype safety, metadata pass-through, auxiliary artifacts</name>
  <files>src/bioimage_mcp/registry/dynamic/adapters/scipy_ndimage.py</files>
  <action>
Update `ScipyNdimageAdapter.execute()` (and minimal helpers) to implement:

1) Named input resolution
- `inputs` arrives as list of `(name, value)` pairs from `tools/base/bioimage_mcp_base/dynamic_dispatch.py`.
- Find the primary image input by key preference: `image` then `input` then first artifact-like entry.
- Load that artifact via `bioio.BioImage` (or equivalent wrapper) to ensure native TCZYX 5D shape. DO NOT use `np.squeeze`.
- Preserve its metadata from the original input dict (axes + physical_pixel_sizes + channel_names when present).

2) Auxiliary array artifacts
- For any additional artifact-like inputs whose names match a missing param key (e.g., `structure`, `footprint`):
  - load them to numpy arrays via `_load_image` or OBJECT_CACHE for `obj://`
  - set `params[name] = loaded_array`.
- Also support params containing artifact-like dicts or `obj://` URIs for those array params.

3) Callable parameters
- For known callable parameter names (start with: `function`, `callback`), if the param value is a string, resolve via `callable_registry.resolve_callable`.

4) Memory safety / dtype
- If the primary input dtype is `uint16` and the array is "large" (define a simple threshold, e.g. `nbytes >= 16 * 1024 * 1024`), cast to `float32` BEFORE calling scipy.
- If the output dtype is `int64` (common in labeling), cast to `int32` or `uint32` before returning to ensure OME-TIFF compatibility.
- Do not force output dtype otherwise (allow scipy defaults).

5) Force new artifact (no in-place)
- If `params` contains an `output` key, drop it unconditionally so scipy cannot write into an input-provided buffer.

6) Output metadata override
- When returning the output artifact dict, include `metadata` keys that the server will merge:
  - `axes`, `dims`, `ndim`, `shape`, `dtype`
  - `physical_pixel_sizes` + `channel_names` pass-through (when present)

Keep default output behavior as file-backed OME-TIFF (as today).
  </action>
  <verify>python -m compileall src/bioimage_mcp/registry/dynamic/adapters/scipy_ndimage.py</verify>
  <done>Adapter can execute with auxiliary artifact inputs and returns output refs whose metadata includes axes and physical_pixel_sizes when provided.</done>
</task>
