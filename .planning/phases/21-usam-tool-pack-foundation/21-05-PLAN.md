<task type="auto">
  <name>Task 2: Implement microsam runtime device selection (auto: CUDA &gt; MPS &gt; CPU) and enforce forced-device availability</name>
  <files>
tools/microsam/bioimage_mcp_microsam/device.py
tools/microsam/bioimage_mcp_microsam/entrypoint.py
  </files>
  <action>
- Create `tools/microsam/bioimage_mcp_microsam/device.py` providing a small, dependency-light API that can be unit-tested without importing torch:
  - `select_device(device: str, *, torch_module=None, strict: bool = True) -> str`
    - `device` accepted values: `auto|cuda|mps|cpu` (treat unknown as programmer error: raise ValueError).
    - When `device == "auto"`: prefer `cuda` if available; else `mps` if available; else `cpu`.
    - When forced to `cuda` or `mps`:
      - If not available and `strict=True`: raise an exception that can be mapped to a stable tool error code (no fallback).
      - If `strict=False`: return the forced value without validating availability (used for meta.describe to avoid breaking discovery on machines without accelerators).
    - If `torch_module is None`: import torch lazily inside the function.
    - Availability checks must be resilient to missing attributes (e.g. no `torch.backends.mps`): treat as unavailable.
- Update `tools/microsam/bioimage_mcp_microsam/entrypoint.py` to:
  - Read `device_pref` from `request.get("tool_config", {}).get("microsam", {}).get("device", "auto")`.
  - Resolve `selected_device = select_device(device_pref, strict=(req_id != "meta.describe"))`.
  - Keep heavy imports out of module import time (only import torch when selecting device).
  - On `strict=True` selection failure: return `ok:false` with structured error:
    - `code: DEVICE_UNAVAILABLE`
    - Message includes the forced device and the selection order.
    - `details` includes at least one entry pointing to `microsam.device` and a remediation hint (e.g. set `microsam.device: auto` or install GPU profile).
  - Do not print/return `selected_device` in normal outputs; it may be logged only in debug/verbose paths if the tool pack already has such logging.
  </action>
  <verify>python -m compileall tools/microsam/bioimage_mcp_microsam >/dev/null</verify>
  <done>
- The microsam tool pack has a deterministic device-resolution function with the required auto preference.
- Forced unavailable devices (cuda/mps) produce a stable `DEVICE_UNAVAILABLE` tool error for non-meta requests.
- meta.describe remains functional even if an accelerator is unavailable.
  </done>
</task>
