  - Supported profiles for microsam: `cpu` and `gpu` only; default is `cpu`.
  - GPU profile semantics:
    - Linux: install CUDA-capable stack (use a post-install step analogous to `_gpu_post_install`, but targeted for the microsam env and a modern `pytorch-cuda` version).
    - macOS Apple Silicon: rely on MPS-capable PyTorch build (no CUDA steps).
    - macOS Intel: print a warning and fall back to CPU profile.
    - Windows: best-effort; if it fails, error message recommends WSL2/Linux/macOS.
- Implement post-env steps for microsam that run even if the env already exists (repair mode):
  1. Run a quick sanity import in the env (micro_sam import).
  2. Execute `tools/microsam/bioimage_mcp_microsam/install_models.py` inside the env to ensure required models are cached.
  3. Capture the script JSON output and persist it to a stable state file under `.bioimage-mcp/` (exact path defined in Plan 21-04).
- UX requirements:
  - On success: print ONE single-line confirmation including `bioimage-mcp-microsam` and the resolved model cache location.
  - On model/bootstrap failure: exit non-zero and print actionable remediation (include env name and cache path if known).
- Update `tests/unit/bootstrap/test_install.py` to cover:
  - `install(tools=["microsam"], profile=None)` defaults to CPU and invokes the model bootstrap step.
  - `install(tools=["microsam"], profile="gpu")` triggers the GPU post-install step on Linux.
  - Failure in the model bootstrap step returns non-zero.
  - Existing non-microsam install tests continue to pass.
