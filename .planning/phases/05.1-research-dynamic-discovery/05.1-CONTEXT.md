# Phase 5.1: Research Dynamic Discovery Standardization - Context

**Gathered:** 2026-01-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Research and standardize dynamic discovery implementations (`meta.list`, `meta.describe`) across packages. Confirm implementation differences, compare them, and standardize to a single discovery method.
</domain>

<decisions>
## Implementation Decisions

### Protocol Strictness
- **Flexible/Duck Typing:** Do not enforce rigid schemas; allow tools to return what they have as long as required fields exist.
- **Pass Through:** Preserve extra/unknown fields in the response (don't strip them).
- **Implicit Versioning:** No explicit `protocol_version` field; evolve naturally.
- **Skip Malformed:** If an item lacks required fields (like `id`), skip it and log a warning; do not fail the entire tool load.

### Discovery Strategy
- **Runtime Introspection:** Discover functions by importing and inspecting objects at runtime (not static AST analysis).
- **Lazy Loading:** Perform discovery only when the tool is accessed or listed (not at startup).
- **Memory Cache:** Cache discovery results in memory for the session; do not persist to disk.
- **Execution:** In-process introspection (import) within the adapter's execution context.

### List Verbosity
- **Summary Only:** `meta.list` returns lightweight IDs and descriptions/names; full metadata requires `describe`.
- **Pagination:** Support cursor-based pagination for large lists.
- **Flat List:** Structure results as a flat list of dotted names (e.g., `scipy.ndimage.filter`), not a nested hierarchy.
- **Server-side Filtering:** Support filtering (name/tag) via query parameters.

### Error Handling
- **Separate Error List:** Return discovery errors in a separate `errors` list in the response body, distinct from valid items.
- **No Auto-Retry:** Do not automatically retry failed discoveries in the background; retry on next request.
- **Restart & Retry:** If the discovery process/subprocess crashes (e.g., segfault), restart it and try once more.

### OpenCode's Discretion
- Exact pagination page size default.
- Specific field names for the "summary" view.
- Format of the error objects in the error list.

</decisions>
