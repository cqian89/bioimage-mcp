  <name>Add meta.list/meta.describe protocol models + parsers</name>
  <files>src/bioimage_mcp/runtimes/meta_protocol.py</files>
  <action>
Create `src/bioimage_mcp/runtimes/meta_protocol.py` with Pydantic v2 models and helper functions:

- Models:
  - `MetaListFunction` (required: `fn_id`, `name`, `summary`; optional: `module`, `io_pattern`, plus allow extra)
  - `MetaListSuccess` / `MetaListError`
  - `MetaDescribeSuccess` / `MetaDescribeError` (require `result.params_schema`, `result.tool_version`, `result.introspection_source`)

- Helper extraction:
  - `extract_result_payload(response: dict) -> dict | None` which supports both shapes:
    - direct: `{ok: true, result: {...}}`
    - worker: `{command: execute_result, ok: true, outputs: {result: {...}}}`
  - `parse_meta_list_result(response: dict) -> list[dict]` returning normalized function entries (skip invalid entries, but do not raise).
  - `parse_meta_describe_result(response: dict) -> dict | None` returning normalized `{params_schema, tool_version, introspection_source}`.

Keep parsing backwards compatible: accept missing optional keys and tolerate extra keys.
  </action>
  <verify>python -m compileall src/bioimage_mcp/runtimes/meta_protocol.py</verify>
  <done>Protocol parsers exist and can parse legacy + worker-shaped payloads without throwing.</done>
</task>

  - Replace the ad-hoc result extraction (`response.get("result")` vs `outputs.result`) with `parse_meta_describe_result()`.
  - Only update cache when parsed result includes a dict `params_schema`.
  - On invalid payloads, log a warning and fall back to manifest schema (current behavior).

Avoid changing public API schemas; this is strictly an internal robustness improvement.
  </action>
  <verify>python -m compileall src/bioimage_mcp/registry/loader.py src/bioimage_mcp/api/discovery.py</verify>
