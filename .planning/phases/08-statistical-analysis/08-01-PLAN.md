Add these new enum members to `src/bioimage_mcp/registry/dynamic/models.py:IOPattern`:
- `TABLE_TO_JSON` (single TableRef/ObjectRef input -> JSON-like output)
- `TABLE_PAIR_TO_JSON` (two TableRef/ObjectRef inputs -> JSON-like output)
- `MULTI_TABLE_TO_JSON` (list of TableRef/ObjectRef inputs -> JSON-like output)
- `PARAMS_TO_JSON` (no artifact inputs -> JSON-like output)

Then extend `src/bioimage_mcp/registry/loader.py:_map_io_pattern_to_ports()`:

Use `NativeOutputRef` for these outputs (format: json). This matches existing adapter behavior (Phase 7 JSON payloads are emitted as `NativeOutputRef`), and avoids incorrectly labeling structured JSON dictionaries as `ScalarRef`.

- `TABLE_TO_JSON` -> inputs: `table` (artifact_type: `[TableRef, ObjectRef]`), outputs: `output` (`NativeOutputRef`)
- `TABLE_PAIR_TO_JSON` -> inputs: `table_a` + `table_b` (artifact_type: `[TableRef, ObjectRef]`), outputs: `output` (`NativeOutputRef`)
- `MULTI_TABLE_TO_JSON` -> inputs: `tables` (artifact_type: `[TableRef, ObjectRef]`, `is_array: true`), outputs: `output` (`NativeOutputRef`)
- `PARAMS_TO_JSON` -> inputs: none, outputs: `output` (`NativeOutputRef`)

Update `tests/unit/registry/test_loader_io_patterns.py` to assert these new patterns map to the expected port names and artifact types.

Keep existing patterns unchanged.
