  <action>
Implement `ScipyStatsAdapter.execute(fn_id, inputs, params, work_dir=None)` to support the wrappers created in discovery.

Input handling (table wrappers):
- Accept inputs as `(name, value)` pairs like other adapters.
- Support `TableRef` (file-backed) and `ObjectRef` (obj://) inputs.
- For `TableRef`: load via pandas (CSV or Parquet). For `ObjectRef`: allow pandas DataFrame/Series or numpy arrays in `OBJECT_CACHE`.
- Select the data vector using `params.get('column')` when provided; otherwise:
  - If there is exactly one numeric column, use it.
  - Else fall back to the first numeric column (deterministic) and include `selected_column` in output payload.
- If `nan_policy == 'omit'`, drop NaNs before calling scipy.

Execution behavior:
- Implement each *_table wrapper by calling the underlying scipy.stats function(s):
  - `describe_table` -> `scipy.stats.describe`
  - `skew_table` -> `scipy.stats.skew`
  - `kurtosis_table` -> `scipy.stats.kurtosis`
  - `ttest_*_table` -> corresponding `scipy.stats.ttest_*`
  - `f_oneway_table` -> `scipy.stats.f_oneway`
  - `ks_2samp_table` -> `scipy.stats.ks_2samp`
- Return a single JSON artifact for all wrappers.

Output shaping:
- Save JSON using the same conventions as Phase 7 (`NativeOutputRef` JSON payload; NaNs -> null; numpy scalars -> native types; tuples/ndarrays -> lists).
- Payload must include at least:
  - `fn_id` (string)
  - `statistic` and `pvalue` for hypothesis tests
  - `n` / `n_a` / `n_b` and `selected_column` metadata for reproducibility

Distribution methods:
- Parse fn_id `scipy.stats.{dist}.{method}` and call the corresponding scipy.stats distribution API.
- Use frozen distributions for consistent parameter application (`dist(*args, loc=..., scale=...)`) with a safe fallback if `scale` is unsupported.
- Return arrays as JSON lists.

Finally, ensure `ScipyAdapter` routes `scipy.stats.*` execution to `ScipyStatsAdapter` even when runtime passes tool-prefixed fn_ids like `base.scipy.stats.*` (strip optional tool prefix before matching and before delegating).
  </action>

    assert out and out[0]["type"] == "NativeOutputRef" and out[0]["format"] == "json"

print("execute ok")
