<task type="auto">
  <name>Task 1: Create remove.py bootstrap module</name>
  <files>src/bioimage_mcp/bootstrap/remove.py</files>
  <action>
Create new module for remove command:

1. **Imports:**
   - subprocess for conda/micromamba commands
   - env_manager.detect_env_manager for manager detection
   - Optional: PersistentWorkerManager for active worker check (if accessible)

2. **Active worker check function:**
   ```python
   def is_tool_active(tool_name: str) -> bool:
       """Check if tool has an active worker process."""
   ```
   - Try to check if there's a running worker for the tool's env_id
   - If PersistentWorkerManager not easily accessible, check for process by env name
   - Return True if active, False if safe to remove
   - On error/uncertainty, return False (allow removal)

3. **Main remove function:**
   ```python
   def remove_tool(tool_name: str, *, yes: bool = False) -> int:
       """Remove a tool environment.
       
       Args:
           tool_name: Tool to remove (e.g., 'cellpose')
           yes: Skip confirmation prompt
           
       Returns:
           0 on success, 1 on error
       """
   ```

4. **Remove logic:**
   a. Validate tool_name is not "base" (base cannot be removed)
   b. Construct env_id: f"bioimage-mcp-{tool_name}"
   c. Check if env exists:
      - Run `{manager} env list` and check for env_id
      - If not found: "Tool 'cellpose' is not installed", return 1
   d. Check if active:
      - If is_tool_active(tool_name): "Cannot remove cellpose: tool is currently running. Stop the server first.", return 1
   e. Confirmation (unless --yes):
      - Print "Remove cellpose? This will delete the conda environment. (y/N): "
      - Read input, require explicit "y" or "yes" (case-insensitive)
      - On "n" or empty: "Cancelled", return 0
   f. Execute removal:
      - Run `{manager} env remove -n {env_id} -y`
      - On success: "Removed cellpose", return 0
      - On failure: Print error, return 1

5. **Output messages:**
   - Keep consistent with CONTEXT.md decisions (brief on success, detailed on error)
   - No colors in first pass
