  <files>src/bioimage_mcp/registry/dynamic/adapters/scipy_ndimage.py</files>
  <action>
Update `ScipyNdimageAdapter.execute()` to special-case `fn_id` targeting `scipy.ndimage.zoom`:

- Read the effective `axes` string from the primary input artifact metadata.
- Read `zoom` from `params` (scipy parameter name).
- Compute a mapping from axis letter -> zoom factor using these rules:
  1) If `zoom` is a scalar: treat it as applying to every axis in `axes`.
  2) If `zoom` is a sequence and `len(zoom) == len(axes)`: zip directly.
  3) Else if `zoom` is a sequence and `len(zoom) == count_of_spatial_axes_in_axes` (axes subset of Z/Y/X in order of appearance): map only those spatial axes.
  4) Otherwise: do not modify physical metadata (leave pass-through unchanged).

- If `metadata_override` has `physical_pixel_sizes`, update only spatial components:
  - New physical size = old physical size / zoom_factor for each of X/Y/Z that is present and has a numeric value.
  - Ignore `T` and `C` for physical size updates.
  - Guard against zero factors (skip update for that axis).

Do NOT add any automatic squeezing or "smart" dimension selection; rely on user preprocessing (Phase 7 context: raw scipy behavior).
  </action>
