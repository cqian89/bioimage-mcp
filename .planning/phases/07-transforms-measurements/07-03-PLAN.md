     - `counts`: JSON artifact with at least `{ "num_features": <int> }`
   - Prevent output overwrites when `work_dir` is provided:
     - Add `filename: str = "output.ome.tiff"` to `_save_image(...)` and use it when computing the `path` under `work_dir`.

   - `center_of_mass`, `sum`, `mean`, `extrema`
   - Normalize outputs into a dict keyed by label id string:
     - If `index` is provided (scalar or list): keys correspond to requested labels.
     - Missing labels: value is `null` (JSON null, not error).
     - If scipy returns a single value (scalar/tuple) for a single index: still wrap it as `{ "<index>": value }`.
     - If scipy returns a list/array for implicit labels: map to `1..N` label ids.

   - Input wiring: these measurements must accept a `labels` input artifact (LabelImageRef) in `inputs`.
     The adapter already supports auxiliary artifacts by name; rely on that to pass `labels=<ndarray>` into scipy.

Always emit measurement results using `_save_json()` (not `_save_scalar()`).

Keep existing generic behavior for all other scipy functions.
