  <action>
Implement cache reuse for trackpy meta.list by reusing the existing IntrospectionCache + discover_functions.

Decision: Recommend Option B (reuse IntrospectionCache) over Option A (bespoke JSON cache).
Rationale:
- Consistent invalidation contract (env lockfile hash gating) across tool packs.
- Reuses existing cache serialization (FunctionMetadata) and avoids duplicating cache logic.
- Uses the trackpy manifest's dynamic_sources instead of hardcoding module lists.

Implementation details:
- Add tools/trackpy/bioimage_mcp_trackpy/dynamic_discovery.py defining a lightweight adapter (name: TrackpyAdapter) with a discover(config) method that:
  - Iterates config["modules"] and calls the existing introspect_module(module_name).
  - Converts each discovered dict to FunctionMetadata:
    - fn_id: from dict
    - name/module: from dict
    - qualified_name: "{module}.{name}" (or derive from fn_id)
    - source_adapter: "trackpy"
    - io_pattern: map string to IOPattern enum (e.g. "image_to_table" -> IOPattern.IMAGE_TO_TABLE)
    - description: empty (trackpy meta.describe remains the source of detailed schema + docstrings)
- Update tools/trackpy/bioimage_mcp_trackpy/entrypoint.py handle_meta_list to:
  - Load tools/trackpy/manifest.yaml into ToolManifest (same pattern as tools.base meta.list).
  - Compute project_root with the same heuristic as src/bioimage_mcp/api/discovery.py:_find_project_root.
  - Build cache_dir as Path.home() / ".bioimage-mcp" / "cache" / "dynamic" / manifest.tool_id
  - Create IntrospectionCache(cache_dir) and call:
    discover_functions(manifest, {"trackpy": TrackpyAdapter()}, cache=cache, project_root=project_root)
  - Convert FunctionMetadata results into the canonical meta.list function dict shape expected by the runtime (matching tools.base fields).
  - Keep tool_version = get_trackpy_version() and introspection_source = "dynamic_discovery".

Non-goal (keep optional unless needed): adding dynamic_source_hash invalidation; if implemented later, it should incorporate a stable fingerprint of dynamic_sources into the cache key.
  </action>
