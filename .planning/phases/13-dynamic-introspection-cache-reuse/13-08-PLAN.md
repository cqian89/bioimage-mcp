<task type="auto">
  <name>Task 1: Invalidate CLI list_tools cache hit when per-tool dynamic cache is missing</name>
  <files>src/bioimage_mcp/bootstrap/list.py</files>
  <action>
Implement a fast-path invariant check before returning `ListToolsCache` payload:

- After `cached_payload = tools_cache.get(fingerprint)` succeeds, scan the cached tool entries and compute which tools are expected to have a per-tool dynamic cache file.
- Treat a tool as "expects dynamic cache" if its cached `introspection_source` list contains `"dynamic_discovery"` (this matches the Phase 13 behavior for trackpy/base).
- For each such tool, check that `Path.home() / ".bioimage-mcp" / "cache" / "dynamic" / tool_id / "introspection_cache.json"` exists.
- If any expected file is missing, do NOT return the cached payload; instead, fall through to the existing cache-miss path (which calls `load_manifests(...)`).
  - Rationale: `load_manifests()` runs `DiscoveryEngine` and will re-trigger tool-pack `meta.list` and re-create `introspection_cache.json` (this is why UAT passes when both caches are deleted).
- Keep behavior unchanged when `BIOIMAGE_MCP_DISABLE_LIST_CACHE=1` (no caching, no extra checks).
- Avoid changing the output schema of `bioimage-mcp list` (no new fields in tool payload).
  </action>
  <verify>pytest tests/unit/bootstrap/test_list_cache.py -q</verify>
  <done>
`bioimage-mcp list` no longer returns a CLI cache-hit result when any required per-tool dynamic cache file is missing; it re-runs discovery and therefore regenerates the missing file.
  </done>
