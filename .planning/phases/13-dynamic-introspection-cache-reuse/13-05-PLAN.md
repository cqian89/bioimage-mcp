Close the Phase 13 UAT warm-cache speed gap for the standalone CLI `bioimage-mcp list` by adding persistent caching for manifest-derived tool summaries and env-manager environment listing.

Purpose: The CLI runs in a fresh process each invocation, so in-memory caches dont help; persistent caching is required for warm runs to be fast.
Output: A small, file-backed cache layer used by `src/bioimage_mcp/bootstrap/list.py` + unit tests that prove cache hits avoid subprocess/manifest reload.

- Implement two caches (can share one JSON file or use two files):

  - List tools cache: stores a manifest fingerprint (based on discovered manifest paths + stat metadata like `mtime_ns`/`size`) plus the computed `tool_details` payload.

- Use `Path.home() / ".bioimage-mcp" / "cache" / "cli"` as the cache directory to align with existing Phase 13 cache conventions.
- Avoid adding new external deps; use stdlib only.
