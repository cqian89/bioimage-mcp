  - `put(lockfile_hash: str, manifest_checksum: str, results: list[dict[str, Any]]) -> None`
- Be resilient to corrupt/missing JSON (return None on read errors; overwrite with fresh structure on write).
- Keep it deterministic (stable JSON output; use indent=2; no random ordering assumptions).

  <action>
Update `DiscoveryEngine._runtime_list()` to use the new persistent cache.

Implementation requirements:
- Only enable caching when BOTH are available:
  - `self.project_root` is not None
  - the env lockfile exists at `${project_root}/envs/{manifest.env_id}.lock.yml`
- Compute `lockfile_hash` as first 16 chars of sha256(lockfile contents) (match tools' behavior).
- Use cache_dir: `Path.home() / ".bioimage-mcp" / "cache" / "dynamic" / manifest.tool_id`.
- On cache hit: return cached list immediately; do NOT call `execute_tool`.
- On cache miss:
  - call `execute_tool` exactly as today
  - parse with `parse_meta_list_result`
  - write parsed list to cache using (lockfile_hash, manifest.manifest_checksum)
- Preserve current failure behavior: if runtime list fails, return [] and do not write.
- Add minimal logging only if needed; do not spam logs in the success path.
  </action>
