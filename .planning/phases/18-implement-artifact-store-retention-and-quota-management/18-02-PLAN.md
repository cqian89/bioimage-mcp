Create `src/bioimage_mcp/storage/manager.py` as the single query/decision layer for retention/quota.

Implement a `StorageManager` that is constructed with (config, sqlite_connection) and provides:
- `get_total_bytes()`:
  - Prefer `registry_state.total_artifact_bytes`.
  - If missing, compute `SUM(size_bytes)` for non-memory artifacts and store/repair the key.
- `get_quota_bytes()` and `get_usage_fraction()`.
- `get_protected_session_ids()`:
  - Find the most recent session(s) by sessions.created_at desc (count = config.storage.protect_recent_sessions).
- `is_session_active(session_id)`:
  - Use sessions.status == 'active' (do not infer from TTL here).
- `list_cleanup_candidates(now_iso, *, limit)`:
  - Select artifacts eligible for deletion by retention rules:
    - created_at older than retention_days
    - pinned == 0
    - storage_type != 'memory'
    - session_id not in protected_session_ids
    - session_id not in active sessions
  - Order by created_at ASC (FIFO).
  - Return rows including ref_id, uri, size_bytes, created_at, storage_type, session_id.

Then update `src/bioimage_mcp/api/artifacts.py` to surface retention timing per artifact:
- Add fields to the response:
  - `created_at` (existing data)
  - `pinned` (from DB if available; default false if missing)
  - `retention_expires_at` (created_at + retention_days)
  - `time_to_cleanup_seconds` (expires_at - now, floor at 0)
- If pinned==true, set retention_expires_at/time_to_cleanup_seconds to null.

Implementation notes:
- Do not add per-artifact listing commands in this phase.
- Keep artifact_info compatible: only add fields, do not remove or rename existing keys.
