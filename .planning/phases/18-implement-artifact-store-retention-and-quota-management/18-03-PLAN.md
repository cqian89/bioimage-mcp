<task type="auto">
  <name>Task 1: Implement cleanup engine (retention + quota) with safety + event logging</name>
  <files>src/bioimage_mcp/storage/cleanup.py</files>
  <action>
Create `src/bioimage_mcp/storage/cleanup.py` implementing the artifact deletion logic.

Core behavior:
- Support two triggers:
  1) retention: delete artifacts older than retention_days
  2) quota: when total_bytes >= quota_bytes * trigger_fraction, delete oldest eligible until total_bytes <= quota_bytes * target_fraction
- Enforce cooldown between runs using `registry_state` keys (e.g., `cleanup_last_run_at`, `cleanup_last_run_reason`):
  - If last_run_at within cooldown_seconds, skip unless `force=True`.
- Skip artifacts that are:
  - pinned
  - storage_type == 'memory'
  - in protected recent sessions
  - in active sessions

Deletion mechanics (file-backed only):
- Implement rename-to-trash before final deletion:
  - For file:// URIs, compute on-disk path.
  - Rename path to `<original>.deleted.<timestamp>` (same directory) before deleting.
  - Then delete (unlink or shutil.rmtree).
  - After successful deletion, delete the artifact DB row (so the AFTER DELETE trigger updates total bytes).
- If the on-disk file/dir is already missing, still delete the DB row (treat as stale registry entry), but record a note count in the event.

Dry-run:
- When dry_run=True, do not rename/delete and do not delete DB rows.
- Return a summary payload with:
  - would_delete_count, would_free_bytes, before_bytes, after_bytes_estimate
  - top-level reason(s) (retention/quota)

Event logging:
- Insert a single row into `cleanup_events` per attempted run (including dry-run), storing summary-only counts.
- Do not log individual artifact IDs in the DB or stdout.

Avoid:
- Any cleanup that runs synchronously inside artifact import/write paths.
- Any directory walking to compute sizes.
  </action>
