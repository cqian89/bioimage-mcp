  <name>Task 1: Extend SQLite schema for retention/quota tracking</name>
  <files>src/bioimage_mcp/storage/sqlite.py</files>
  <action>
Add schema support for retention/quota management using the existing init_schema + PRAGMA migrations pattern.

Implement the following in `init_schema(conn)`:
- Add columns to `artifacts` (ALTER TABLE guarded by PRAGMA table_info checks):
  - `session_id TEXT` (nullable)
  - `pinned INTEGER NOT NULL DEFAULT 0` (treat as boolean)
  - `last_accessed_at TEXT` (nullable, ISO timestamp)
- Add indexes (CREATE INDEX IF NOT EXISTS) for cleanup query performance:
  - artifacts(created_at)
  - artifacts(session_id)
  - artifacts(pinned)
- Add trigger-based tracking of total bytes in `registry_state` key `total_artifact_bytes`:
  - AFTER INSERT on artifacts: add NEW.size_bytes
  - AFTER DELETE on artifacts: subtract OLD.size_bytes
  - Do not track memory artifacts (storage_type == 'memory') in totals. Enforce this by making triggers conditional.
- Add a `cleanup_events` table for summary-only cleanup logging:
  - id INTEGER PRIMARY KEY AUTOINCREMENT
  - started_at TEXT NOT NULL
  - ended_at TEXT NOT NULL
  - reason TEXT NOT NULL
  - dry_run INTEGER NOT NULL
  - deleted_count INTEGER NOT NULL
  - freed_bytes INTEGER NOT NULL
  - before_bytes INTEGER NOT NULL
  - after_bytes INTEGER NOT NULL
  - notes_json TEXT (nullable)
- Add a one-time "backfill/repair" step:
  - Ensure `registry_state.total_artifact_bytes` exists.
  - Recompute from DB (SUM(size_bytes) WHERE storage_type != 'memory') and set the key if missing or clearly invalid.

Avoid:
- Any filesystem size scanning.
- Any behavior that blocks tool execution paths.
