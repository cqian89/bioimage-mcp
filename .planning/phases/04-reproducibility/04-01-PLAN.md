  <action>
Add a private method `_validate_overrides` to `SessionService` that:
1. Import `jsonschema` at the top: `from jsonschema import validate, ValidationError`
   Note: jsonschema is already a transitive dependency via pydantic; verify with import check.
2. Takes `params_overrides`, `step_overrides`, and the `WorkflowRecord` as arguments
3. For each function in `params_overrides`:
   - Get the function descriptor using `discovery_service.describe_function(fn_id=fn_id)`
   - Extract `params_schema` from the descriptor
   - Validate the override params against the schema using `jsonschema.validate(instance=override_params, schema=params_schema)`
4. For each step override in `step_overrides`:
   - Map `step:{idx}` to the function at that step index
   - Validate any `params` in the override against that function's schema
5. Collect all validation errors and return a list of `ErrorDetail` objects with:
   - `path`: e.g., `/params_overrides/cellpose.base.segment/diameter`
   - `expected`: from schema (type or constraint)
   - `actual`: the invalid value
   - `hint`: from jsonschema error message

Handle case where function is not found gracefully (skip validation, will fail later during execution).
