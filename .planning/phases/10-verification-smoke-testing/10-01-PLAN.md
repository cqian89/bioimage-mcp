In `tests/smoke/test_smoke_scipy_submodules.py`, implement two parametrized matrices:

1) Minimal coverage (fast CI):
- Markers: `@pytest.mark.smoke_minimal`, `@pytest.mark.uses_minimal_data`, `@pytest.mark.requires_env("bioimage-mcp-base")`, `@pytest.mark.anyio`.
- One representative function per submodule:
  - ndimage: `base.scipy.ndimage.gaussian_filter` (image -> image)
  - stats: `base.scipy.stats.describe_table` (table -> json)
  - spatial: `base.scipy.spatial.distance.cdist` (table pair -> file)
  - signal: `base.scipy.signal.periodogram` (table -> table)

2) Full coverage (broad matrix):
- Markers: `@pytest.mark.smoke_full`, `@pytest.mark.uses_minimal_data`, `@pytest.mark.requires_env("bioimage-mcp-base")`, `@pytest.mark.anyio`.
- 6-10 representative functions per submodule (parametrized so pytest reports all failures, no early exit):
  - ndimage (IMAGE_TO_IMAGE + image measurements):
    - `base.scipy.ndimage.gaussian_filter`, `base.scipy.ndimage.median_filter`, `base.scipy.ndimage.uniform_filter`,
      `base.scipy.ndimage.binary_dilation`, `base.scipy.ndimage.binary_erosion`, `base.scipy.ndimage.label`
  - stats (wrappers + one distribution):
    - `base.scipy.stats.mean_table`, `base.scipy.stats.skew_table`, `base.scipy.stats.kurtosis_table`,
      `base.scipy.stats.ttest_1samp_table`, `base.scipy.stats.ttest_ind_table`, `base.scipy.stats.f_oneway_table`,
      `base.scipy.stats.norm.cdf`
  - spatial:
    - `base.scipy.spatial.distance.cdist`, `base.scipy.spatial.Voronoi`, `base.scipy.spatial.Delaunay`,
      `base.scipy.spatial.cKDTree`, `base.scipy.spatial.cKDTree.query`
  - signal:
    - `base.scipy.signal.periodogram`, `base.scipy.signal.welch`

For each fn_id, prepare the correct inputs:
- ndimage: image input only; pick safe params (e.g., sigma=1.0 for gaussian, size=3 for median/uniform).
- stats wrappers: two small CSV tables with one numeric column named `val`; pass params `{"column": "val"}` where applicable.
- `ttest_1samp_table`: pass params with `popmean` (choose deterministic value).
- `f_oneway_table`: provide 3 table inputs using the MULTI_TABLE pattern (ports should follow the loader mapping).
- `norm.cdf`: call with params only (e.g., x=[0.0, 1.0], loc=0, scale=1).
- spatial:
  - `cdist`: two point tables with numeric columns `x,y`.
  - `cKDTree` then `cKDTree.query`: build returns ObjectRef; feed it into query along with a query table.
- signal:
  - `periodogram`/`welch`: pass a table input with `signal` column and params like `{"column": "signal", "fs": 10.0}`.

Keep full suite deterministic (no random sampling without a fixed seed). Use pytest parametrization so failures are reported per function.
  </action>
