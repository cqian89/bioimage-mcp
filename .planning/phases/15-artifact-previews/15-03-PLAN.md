Ensure there's an appropriate error class or constant for this code that can be used in error responses.

**In artifacts.py**, extend artifact_info for ObjectRef:

1. For ObjectRef/GroupByRef/FigureRef/AxesRef/AxesImageRef artifacts:
   - Add `native_type` field from `ref.python_class`
   - This is the fully qualified class name (e.g., "cellpose.models.Cellpose")

2. Add `object_preview` field:
   - For memory artifacts with `_simulated_path` in `ref.metadata`:
     - Try to load the pickled object
     - Get `repr()` of the object
     - Truncate to 500 chars with "..." if longer
   - For true memory artifacts (no `_simulated_path` in `ref.metadata`):
     - Return placeholder: "In-memory object (not serialized)"
   - On any exception, omit the field

3. Add session-scoped validation for memory artifacts:
   - If `ref.storage_type == "memory"` and `_simulated_path` exists but cannot be read
   - Return error response:
     ```python
     {
         "error": {
             "code": "OBJECT_REF_EXPIRED",
             "message": f"ObjectRef {ref_id} is no longer accessible",
             "details": [{
                 "path": "/ref_id",
                 "hint": "Object references are session-scoped and expire when the session ends"
             }]
         }
     }
     ```

Note: Missing `ref_id` still returns NOT_FOUND. OBJECT_REF_EXPIRED only triggers when metadata exists and file-backed payload is unavailable.

**In server.py**, keep the artifact_info tool signature in sync (no new params here unless needed).
