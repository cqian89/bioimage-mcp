  <action>
**In errors.py**, add new error code:

```python

OBJECT_REF_EXPIRED = "OBJECT_REF_EXPIRED"
```

Ensure there's an appropriate error class or constant for this code that can be used in error responses.

**In artifacts.py**, extend artifact_info for ObjectRef:

1. For ObjectRef/GroupByRef/FigureRef/AxesRef/AxesImageRef artifacts:
   - Add `native_type` field from `ref.python_class`
   - This is the fully qualified class name (e.g., "cellpose.models.Cellpose")

2. Add `object_preview` field:
   - For memory artifacts with `_simulated_path`:
     - Try to load the pickled object
     - Get `repr()` of the object
     - Truncate to 500 chars with "..." if longer
   - For true memory artifacts (no _simulated_path):
     - Return placeholder: "In-memory object (not serialized)"
   - On any exception, omit the field

3. Add session-scoped validation:
   - Check if ObjectRef is still valid (exists in memory store)
   - If expired, return error response:
     ```python
     {
         "error": {
             "code": "OBJECT_REF_EXPIRED",
             "message": f"ObjectRef {ref_id} is no longer accessible",
             "details": [{
                 "path": "/ref_id",
                 "hint": "Object references are session-scoped and expire when the session ends"
             }]
         }
     }
     ```

Note: For now, ObjectRef validation is soft - we check if it exists but don't block on missing objects since the memory store might not have the live object.
  </action>

   - Scan function inputs for ObjectRef types

     - Check if FunctionInputs specifies an ObjectRef type
     - Check hints or type annotations for expected class
     - Add `"x-native-type": "module.ClassName"` to that property in params_schema

2. The x-native-type annotation should come from:
   - FunctionDef.inputs[].type if it's an ObjectRef variant
   - FunctionDef.hints if available
   - Parameter type annotation if it specifies ObjectRef[SomeClass]

3. Example output:
   ```json
   {
     "properties": {
       "model": {
         "type": "string",
         "description": "Cellpose model to use",
         "x-native-type": "cellpose.models.Cellpose"
       }
     }
   }
   ```

4. If no specific type is known, use "object" as x-native-type.

Note: This enhancement may require checking the manifest or hints for ObjectRef type information. If ObjectRef parameters aren't clearly identifiable, add the annotation only when explicit type info is available.
