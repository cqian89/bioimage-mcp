  <name>Task 1: Add LabelImageRef colormap and region metadata</name>
  <files>src/bioimage_mcp/artifacts/preview.py, src/bioimage_mcp/api/artifacts.py</files>
  <action>
**In preview.py**, add:

1. `TAB20_RGB` constant - list of 20 RGB tuples from matplotlib's tab20 colormap:
   ```python
   TAB20_RGB = [
       (31, 119, 180), (174, 199, 232), (255, 127, 14), (255, 187, 120),
       (44, 160, 44), (152, 223, 138), (214, 39, 40), (255, 152, 150),
       (148, 103, 189), (197, 176, 213), (140, 86, 75), (196, 156, 148),
       (227, 119, 194), (247, 182, 210), (127, 127, 127), (199, 199, 199),
       (188, 189, 34), (219, 219, 141), (23, 190, 207), (158, 218, 229)
   ]
   ```

2. `apply_tab20_colormap(label_array: np.ndarray) -> np.ndarray`
   - Input: 2D integer array of labels
   - Output: RGBA array (H, W, 4) with uint8 values
   - Map label value N to TAB20_RGB[(N-1) % 20]
   - Label 0 (background) remains transparent (alpha=0)
   - All other labels get alpha=255

3. `get_label_metadata(label_array: np.ndarray) -> dict`
   - Use scipy.ndimage.center_of_mass for centroids
   - Return `{"region_count": int, "centroids": list[tuple[float, float]]}`
   - Skip label 0 in count and centroids
   - Handle empty label array gracefully

4. `generate_label_preview(path: Path, ...) -> dict | None`
   - Similar to generate_image_preview but:
   - After reduce_to_2d, apply apply_tab20_colormap
   - Encode RGBA as PNG (Pillow handles this)
   - Add region_count and centroids to return dict

**In artifacts.py**, update artifact_info:

1. For LabelImageRef (in addition to BioimageRef handling):
   - Use `generate_label_preview` instead of `generate_image_preview`
   - Add `image_preview.region_count` and `image_preview.centroids` to response
  </action>
  <verify>

   - Load CSV using pandas with nrows=preview_rows

   - Generate markdown table using df.to_markdown(index=False)
   - Extract dtypes as dict: {column_name: str(dtype)}
   - Return:
     ```python
     {
         "table_preview": markdown_string,
         "dtypes": {"col1": "int64", "col2": "object", ...},
         "total_rows": int,  # from metadata or count
         "total_columns": int
     }
     ```
   - Return None on failure

   - `preview_rows: int = 5`
   - `preview_columns: int | None = None`

2. For TableRef artifacts:
   - Call `generate_table_preview()` with parameters
   - Add `table_preview`, `dtypes`, `total_rows`, `total_columns` to response
   - Only generate when any table preview parameter is explicitly provided OR when text_preview_bytes is requested (backward compat)
