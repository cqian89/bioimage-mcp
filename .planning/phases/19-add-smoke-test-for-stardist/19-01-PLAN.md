  <name>Task 3: Add StarDist equivalence smoke test (MCP vs native)</name>
  <files>
tests/smoke/test_equivalence_stardist.py
  </files>
  <action>
- Create `tests/smoke/test_equivalence_stardist.py` mirroring the structure of `tests/smoke/test_equivalence_cellpose.py`.
- Mark the test with:
  - `@pytest.mark.smoke_full`
  - `@pytest.mark.uses_minimal_data`
  - `@pytest.mark.requires_env("bioimage-mcp-stardist")` (required for marker enforcement)
  - `@pytest.mark.requires_stardist` (new marker alias, optional but should be present)
  - `@pytest.mark.anyio`
- Test flow (single test):
  1. Run native baseline via `native_executor.run_script(env_name="bioimage-mcp-stardist", script_path=.../stardist_baseline.py, args=[...])` to produce:
     - an input OME-TIFF path
     - a native labels OME-TIFF path
  2. Start MCP execution via `live_server`:
     - `base.io.bioimage.load` with params `{path: <input_path>}` to get a BioImageRef.
     - `stardist.models.StarDist2D.from_pretrained` with params `{name: "2D_versatile_fluo"}` to get an ObjectRef; wrap this MCP call in a 3-attempt retry loop with backoff.
     - `stardist.models.StarDist2D.predict_instances` with inputs `{model: <ObjectRef>, image: <BioImageRef>}` and no params (use defaults).
  3. Structural assertions (must):
     - `outputs["labels"].type == "LabelImageRef"` and `format == "OME-Zarr"` and `path` exists.
     - `outputs["details"].type == "NativeOutputRef"` and JSON file exists and includes keys like `coord` and `points`.
     - Labels shape matches the 2D input, and dtype is an integer type (uint16/uint32).
     - Label count (non-zero unique ids) is within a broad sanity range (avoid brittle exact counts).
  4. Equivalence assertion (must):
     - Load MCP labels (OME-Zarr directory) and native labels (OME-TIFF) with `bioio.BioImage`, convert to numpy arrays.
     - Compare with `helper.assert_labels_equivalent(..., iou_threshold=0.95)`.
  </action>
  <verify>
pytest --smoke-full -q tests/smoke/test_equivalence_stardist.py
  </verify>
