# Milestone v0.4.0: Unified Introspection Engine

**Status:** ✅ SHIPPED 2026-02-04
**Phases:** 11-20
**Total Plans:** 40

## Overview

Milestone v0.4.0 delivers a robust, high-performance introspection engine for Bioimage-MCP. By implementing an AST-first discovery pipeline with isolated runtime fallback, the system achieves fast, import-free tool discovery while maintaining 100% schema fidelity. This milestone also standardizes OME-Zarr as the primary interchange format, enables multimodal artifact previews for LLMs, integrates StarDist segmentation, and implements automated storage management.

## Milestone Summary

**Key Decisions:**
- Decision: AST-first discovery using `griffe` with on-demand runtime fallback (Rationale: maximize performance while maintaining fidelity).
- Decision: Use OME-Zarr as default artifact format (Rationale: support N-D data with rich metadata and efficient partial reads).
- Decision: Implement Base64-encoded image previews in `artifact_info` (Rationale: enable direct visual verification for multimodal LLMs).
- Decision: Link artifacts to sessions in SQLite for precise quota/retention management (Rationale: clean up temporary data while preserving useful artifacts).
- Decision: Tiered smoke tests (minimal/pr/extended) (Rationale: balance developer velocity with comprehensive verification).

**Issues Resolved:**
- Stabilized tool discovery across all tool packs (base, cellpose, trackpy, stardist).
- Fixed significant metadata loss in artifact roundtrips by standardizing on OME-Zarr.
- Greatly accelerated `bioimage-mcp list` via persistent schema and manifest caching.

**Technical Debt Incurred:**
- Deep validation of OME-Zarr metadata is currently a placeholder.
- Directory-backed materialization refinement for complex Zarr structures.
- `test_equivalence_phasorpy.py` remains in a known-failing state in the extended tier.
