# Requirements Archive: v0.4.0 Unified Introspection Engine

**Status:** ✅ COMPLETED 2026-02-04
**Milestone:** v0.4.0

## v1 Requirements

Requirements for this milestone. All have been validated.

### Introspection Pipeline

- [x] **INT-01**: Agent can discover tools with params_schema derived from AST without importing tool code. (Validated in Phase 12)
- [x] **INT-02**: Engine falls back to isolated runtime introspection when AST signatures are missing or dynamic. (Validated in Phase 12)
- [x] **INT-03**: MCP tools/list and meta.describe share a single introspection source so metadata is consistent. (Validated in Phase 12)

### Schema Emission

- [x] **SCHEMA-01**: Agent receives params_schema derived from type hints and docstrings with required fields populated. (Validated in Phase 12)
- [x] **SCHEMA-02**: params_schema excludes artifact ports; artifact I/O remains separate from parameters. (Validated in Phase 12)
- [x] **SCHEMA-03**: meta.describe includes tool_version and introspection_source for each function. (Validated in Phase 12)

### Environment Readiness

- [x] **ENV-01**: Operator can see readiness diagnostics for missing tool environments or packages with remediation. (Validated in Phase 12/CLI)

### Caching

- [x] **CACHE-01**: Schema cache persists across server restarts. (Validated in Phase 13)
- [x] **CACHE-02**: Cache invalidates on tool_version, env lock hash, or source hash changes. (Validated in Phase 13)

### Overlays and Patches

- [x] **OVERLAY-01**: Tool authors can apply overlays to rename/drop/annotate parameters without code changes. (Validated in Phase 12)
- [x] **OVERLAY-02**: Overlay precedence is deterministic across built-in, manifest, and user overrides. (Validated in Phase 12)

### Metadata and Diagnostics

- [x] **META-01**: Functions report full module paths with stable fn_id metadata. (Validated in Phase 12)
- [x] **META-02**: callable_fingerprint is recorded and exposed for cache/diagnostic use. (Validated in Phase 12)
- [x] **DIAG-01**: Diagnostics report missing docs, runtime fallback usage, and patch application. (Validated in Phase 12)

## Outcomes

- **Validated**: All v1 requirements were met and verified through automated smoke and contract tests.
- **Adjusted**: None.
- **Dropped**: None.
