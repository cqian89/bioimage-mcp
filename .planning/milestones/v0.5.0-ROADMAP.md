# Milestone v0.5.0: Interactive Annotation (micro-sam)

This milestone integrates the upstream `micro_sam` ecosystem into Bioimage-MCP:

- Phase 22 exposes the `micro_sam` Python library API to MCP `run()` with IDs `micro_sam.<submodule>.<callable>` (excluding the napari GUI module).
- Phase 23 exposes the interactive napari annotators (`micro_sam.sam_annotator`).
- Phase 24 makes the end-to-end workflow practical via caching/state artifacts and verification.

## Phases

### Phase 21: Tool Pack Foundation (Complete)
- Isolated conda environment `bioimage-mcp-microsam`.
- Automatic model downloading and GPU device selection.

### Phase 22: Headless API Integration
- Dynamic discovery of `micro_sam.*` via `Introspector` (AST + docstrings).
- Exposure of all `micro_sam` submodules except `micro_sam.sam_annotator`.
- Artifact boundary bridging where feasible (`BioImageRef`/`LabelImageRef` <-> arrays; state via `ObjectRef`).

### Phase 23: Interactive Bridge
- Integration of `micro_sam.sam_annotator` for local/remote GUI launching.
- Data transfer from MCP artifacts to Napari layers.

### Phase 24: Session Management & Optimization
- Persistence of predictors and embeddings across MCP calls.
- Standardized storage for computed embeddings.

## Requirements Covered
- **USAM-01:** Basic SAM inference.
- **USAM-02:** Prompt-based segmentation (points, boxes).
- **USAM-03:** Automatic instance segmentation.
- **USAM-04:** Interactive Napari bridge.
- **USAM-05:** GPU acceleration support.
- **USAM-06:** Model management (LM/EM generalists).

## Acceptance Criteria
- `bioimage-mcp list` exposes `micro_sam.*` callables with stable IDs and schemas.
- Phase 22 excludes only `micro_sam.sam_annotator.*`.
- Headless segmentation works end-to-end with `BioImageRef` artifacts.
- Napari can be launched with data pre-populated from an MCP session and exports committed labels back as artifacts.
