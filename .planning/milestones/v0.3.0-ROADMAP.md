# Milestone v0.3.0: Scipy Integration

**Status:** ✅ SHIPPED 2026-01-27
**Phases:** 5.1-10
**Total Plans:** 25

## Overview

Milestone v0.3.0 integrates the SciPy ecosystem into Bioimage-MCP using a **Dynamic Adapter Pattern**. This enables AI agents to perform advanced N-D image processing, statistical analysis, and spatial modeling through a single, secure interface. The milestone focuses on high-fidelity schema generation via `numpydoc` and memory-safe execution patterns.

## Milestone Summary

**Decimal Phases:**
- Phase 5.1: Research Dynamic Discovery Standardization (inserted after Phase 5 for protocol alignment)

**Key Decisions:**
- Decision: Use `datasets/smoke_tmp` for test CSVs (Rationale: live server read access within allowed paths)
- Decision: Standardize on NativeOutputRef for stats JSON outputs (Rationale: stable structured payloads)
- Decision: Automatic float32 promotion for filters/transforms (Rationale: avoid uint16 overflow, ensure parity)
- Decision: Use docstring-parser for scientific docstrings (Rationale: higher-fidelity schema extraction)
- Decision: Normalize stats outputs to JSON artifacts (Rationale: consistent downstream consumption)

**Issues Resolved:**
- Standardized meta.list/meta.describe protocol and persisted discovery metadata end-to-end.
- Fixed KDTree lifecycle gaps (ref_id) and signal periodogram input handling.
- Preserved physical pixel sizes/channel names in OME-TIFF roundtrips.

**Issues Deferred:**
- trackpy auto-generated schemas missing parameter descriptions.
- base.phasorpy schema type mismatch for `n_components`.
- Contract tests should ignore non-manifest YAML files (e.g., `scipy_ndimage_blacklist.yaml`).

**Technical Debt Incurred:**
- tools.base subprocess discovery still needs parity checks for meta.list output.
- trackpy meta.describe omits tool_version in some paths.
