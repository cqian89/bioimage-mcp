# Milestone v0.2.0: Foundation

**Status:** ✅ SHIPPED 2026-01-25
**Phases:** 1-5
**Total Plans:** 17

## Overview

Established bioimage-mcp foundation with core libraries. This milestone delivered the core runtime, tool management CLI, zero-copy artifact system, reproducibility engine, and the first major integration (Trackpy).

## Phases

### Phase 1: Core Runtime

**Goal**: System can reliably spawn and manage persistent worker processes in isolated environments.
**Focus**: Process lifecycle, IPC, GPU detection.

Plans:

- [x] 01-01: Add MPS (Apple Silicon) GPU detection to complete CORE-02

**Details:**
Delivered core server infrastructure capable of managing isolated Conda environments. Implemented robust NDJSON communication over stdio and unified GPU detection for NVIDIA (CUDA) and Apple Silicon (MPS).

### Phase 2: Tool Management

**Goal**: User can manage the lifecycle of tool environments via CLI.
**Focus**: Installation, verification, removal.

Plans:

- [x] 02-01: Implement `list` CLI command
- [x] 02-02: Refactor `install` CLI command
- [x] 02-03: Implement `remove` CLI command
- [x] 02-04: Refactor `list` for filesystem discovery

**Details:**
Delivered full CLI suite (`install`, `list`, `remove`, `doctor`) enabling users to easily manage tool environments. Implemented dynamic discovery of installed tools.

### Phase 3: Data & Artifacts

**Goal**: System enables zero-copy data passing and artifact management.
**Focus**: File paths, `mem://` protocol, `bioio` integration.

**Details:**
Implemented artifact-based I/O allowing tools to exchange file paths and memory references (`mem://`), avoiding heavy serialization overhead.

### Phase 4: Reproducibility

**Goal**: Users can record and reproduce analysis sessions with validation, error handling, and resume capability.
**Focus**: Session recording, workflow export, production-ready replay.

Plans:

- [x] 04-01: Add override validation using jsonschema
- [x] 04-02: Add version mismatch warnings and environment checks
- [x] 04-03: Add step progress reporting and tool message surfacing
- [x] 04-04: Add missing input handling, resume capability, and error summaries

**Details:**
Delivered comprehensive reproducibility engine. System records all tool inputs, outputs, and versions. Users can export sessions to reproducible workflow files and replay them with validation and error handling.

### Phase 5: Trackpy Integration

**Goal**: Integrate trackpy particle tracking library as a tool pack with full API coverage and live smoke tests.
**Depends on**: Phase 4
**Focus**: Dynamic introspection, API coverage, test data, smoke testing.

Plans:

- [x] 05-02: Implement TrackpyAdapter for dynamic introspection
- [x] 05-03: Add smoke tests with synthetic data fixtures
- [x] 05-04: Fix core stability issues (Gap Closure)
- [x] 05-05: Fix tool entrypoints (Gap Closure)
- [x] 05-06: End-to-end verification via smoke tests
- [x] 05-07: Doctor: treat missing conda-lock as warning (Gap Closure)

Successfully integrated `trackpy` v0.7. Implemented dynamic introspection to automatically discover and expose functions. Verified with extensive smoke tests and synthetic data.

## Milestone Summary

**Key Decisions:**

- **Hub-and-Spoke Architecture**: Persistent workers for performance (Verified)
- **Artifact I/O**: JSON too slow for images; pass paths (Verified)
- **Native GPU**: Docker GPU support complex for end-users; use Conda (Verified NVIDIA/MPS)
- **NDJSON over stdio**: Robust, streamable IPC (Verified)
- **SQLite persistence**: Lightweight, file-based storage (Verified)

**Issues Resolved:**

- Implemented unified GPU detection (CUDA/MPS)
- Solved dynamic tool discovery and introspection
- Established reproducibility pattern with provenance tracking

**Technical Debt Incurred:**

- `DATA-02`: True zero-copy deferred (using file simulation)
- Interactive Input (`INTR-01`) deferred to v2
- Streaming Progress (`INTR-02`) deferred to v2
