# PhasorPy Adapter API Contract

This document specifies the API contracts for the `phasorpy-adapter` feature, ensuring that PhasorPy functions are correctly exposed and executable through the bioimage-mcp MCP server.

## 1. Discovery Contract

The adapter must expose PhasorPy functions through the existing MCP discovery mechanism. All functions from `phasorpy` v0.9+ that follow the standard input/output patterns should be discoverable.

### list_tools
The `list_tools` response includes summaries of PhasorPy functions.

```yaml
list_tools:
  response:
    tools:
      - id: "phasorpy.phasor.phasor_from_signal"
        summary: "Compute phasor from signal"
        tags: ["phasor", "transform", "flim"]
      - id: "phasorpy.phasor.phasor_transform"
        summary: "Transform phasor coordinates"
        tags: ["phasor", "transform"]
      # ... 50+ more functions
```

### describe_function
The `describe_function` tool returns the full Pydantic-validated schema for a specific function.

```yaml
describe_function:
  input:
    fn_id: "phasorpy.phasor.phasor_from_signal"
  response:
    fn_id: "phasorpy.phasor.phasor_from_signal"
    name: "phasor_from_signal"
    description: "Compute phasor coordinates from time-resolved signal."
    io_pattern: "signal_to_phasor"
    inputs:
      - name: "signal"
        artifact_type: "BioImageRef"
        required: true
        description: "FLIM signal array with decay dimension"
    outputs:
      - name: "mean"
        artifact_type: "BioImageRef"
      - name: "real"
        artifact_type: "BioImageRef"  
      - name: "imag"
        artifact_type: "BioImageRef"
    params_schema:
      type: object
      properties:
        axis:
          type: integer
          default: -1
          description: "Axis of decay bins"
        harmonic:
          type: integer
          default: 1
```

## 2. Execution Contract

All PhasorPy functions are executed via the `run_function` tool. The adapter handles the conversion between bioimage-mcp artifacts and PhasorPy-compatible NumPy arrays or metadata.

```yaml
run_function:
  input:
    fn_id: "phasorpy.phasor.phasor_from_signal"
    inputs:
      signal: 
        ref_id: "abc123"  # BioImageRef pointing to FLIM data
    params:
      axis: -1
      harmonic: 1
  response:
    ok: true
    outputs:
      mean:
        ref_id: "def456"
        type: "BioImageRef"
        format: "OME-TIFF"
      real:
        ref_id: "ghi789"
        type: "BioImageRef"
        format: "OME-TIFF"
      imag:
        ref_id: "jkl012"
        type: "BioImageRef"
        format: "OME-TIFF"
```

## 3. PlotRef Contract

PhasorPy plotting functions return a `PlotRef` artifact, which contains metadata specific to visualization.

```yaml
run_function:
  input:
    fn_id: "phasorpy.plot.plot_phasor"
    inputs:
      real:
        ref_id: "ghi789"
      imag:
        ref_id: "jkl012"
    params:
      colormap: "viridis"
  response:
    ok: true
    outputs:
      plot:
        ref_id: "mno345"
        type: "PlotRef"
        format: "PNG"
        metadata:
          width_px: 800
          height_px: 600
          dpi: 100
          plot_type: "phasor_plot"
```

## 4. Error Contract

The adapter must catch and translate PhasorPy-specific errors or environment issues into standard bioimage-mcp error responses.

```yaml
run_function:
  input:
    fn_id: "phasorpy.phasor.phasor_from_signal"
    inputs:
      signal:
        ref_id: "invalid_ref"
    params: {}
  response:
    ok: false
    error:
      code: "ARTIFACT_NOT_FOUND"
      message: "Input artifact 'invalid_ref' not found"
```

## 5. Success Criteria

The following criteria must be met for the PhasorPy adapter to be considered compliant:

- **SC-001**: ≥50 phasorpy functions discovered and correctly described.
- **SC-002**: End-to-end workflow (Load → Transform → Calibrate → Plot) completes in < 30 seconds for standard datasets.
- **SC-003**: Core server remains stable even if vendor-provided function definitions are malformed.
- **SC-004**: 100% of plotted artifacts (PNG/SVG) generated by PhasorPy are accessible via the `get_artifact` tool.
